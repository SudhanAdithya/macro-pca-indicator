Macro-PCA Recession Indicator: Methodology & Implementation

This project implements a robust macroeconomic signaling engine designed to identify Economic Recession Regimes. By applying Principal Component Analysis (PCA) to a broad panel of high-frequency indicators, we extract a unified “latent” signal that removes individual variable noise and captures synchronized economic contractions.

1. Methodology: The PCA Engine

The core of this project is the Synthesis of Disparate Data. Instead of looking at a single number (like GDP), we look at the covariance of the entire “economic body.”

10-Variable Macro Panel

We ingest and process data from the FRED API across three pillars: 1. Real Activity: Industrial Production, Mfg New Orders, Capacity Utilization. 2. Labor Market: Unemployment Rate (inverted signal), Real Personal Income. 3. Consumption: Retail Sales, Housing Starts. 4. Financial Proxies: S&P 500 Log Returns, Yield Curve (10Y-2Y), High Yield Spreads.

Signal Extraction

  • Standardization: Indicators are transformed into % MoM or Log Returns and standardized into Z-scores.
  • Factor Extraction: PCA identifies the “Principal Component 1” (PC1), which explains ~32.5% of the variance in the US economy.
  • Interpretation: When PC1 is positive, the economy is expanding. When PC1 crosses below 0.00, it signals a synchronized slowdown.

2. Recession Regime Identification

The secondary stage of the project maps the PC1 signal into actionable regimes.

Validation Against NBER History

We calibrate our model using the 2000-2026 period. By comparing our “Estimated Regimes” against the official NBER Recession Dates, the model achieved: - Recall (Coverage): 96.4% — The model successfully captured nearly all historical recession months. - Zero-Point Calibration: Using 25 years of data allows us to set the “Normal” threshold at exactly 0.00.

3. Application: Dynamic Portfolio Optimization

As a practical application of the regime engine, we developed a tactical asset allocation strategy.

Evolution from Binary to Continuous

Initially, the project used a binary “Flip Switch” (Stocks vs. Bonds). We evolved this into a Smooth Logistic Mapping (Rt). - High-Precision Scaling: Instead of 60/40 bundles, the strategy calculates exact decimal weights (e.g., 68.32% Stocks) based on the micro-movements of the macro signal. - Boom Scenario: The model is optimized to allow up to 90% equity exposure during periods of extreme PC1 strength.

4. Conclusion

The primary value of this project lies in its ability to quantify economic health objectively. By filtering out the noise of individual reports and focusing on synchronized factor movements, the model provides a reliable, data-driven early warning system for recession regimes.
